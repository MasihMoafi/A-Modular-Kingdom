# Elpis Product Completion Gate

This branch is complete only when all three product pillars are implemented and verified together:

1. Distinctive Elpis UI/UX: amber identity, persistent provider/model/context/memory status, context and memory surfaces, and visible eviction/continuity feedback.
2. Memory and context: lifecycle-aware context cleaning with evidence pointers plus durable-memory archive behavior and focused tests.
3. Model routing: OpenAI, OpenRouter, Anthropic Claude, and Google Gemini provider paths with explicit authentication boundaries and provider-aware selection.

The branch must pass formatting, focused tests, the Elpis binary build, executable identity checks, and updated source-of-truth documentation before merge.
