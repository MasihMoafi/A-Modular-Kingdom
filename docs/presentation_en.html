Core Component

RAG Pipeline (v3)

Hybrid Retrieval Stack

1Ingest + hash files (700 tok chunks)
2Embed (Ollama / ST / OpenAI / Cohere)
3Store in Qdrant Cloud
4Query: BM25 + vector, merged
5Rerank (Cohere, top-5)
6Return JSON with metadata

Key Properties

  • Incremental — only re-indexes changed files
  • Isolated — unique Qdrant collection per path
  • Concurrent — Qdrant Cloud, no file locks
  • Fast — 0.02s warm query (15x vs v2)
  • Supports .py .md .txt .ipynb

Hierarchical Memory System

Persistent rules and preferences that apply across all projects. Stored in ~/.modular_kingdom.

Survives project deletion. Set once, enforced everywhere.

Built on Mem0 + ChromaDB. Scoped routing via ScopedMemoryManager.

Design Decision

Query Routing: Rule-Based, Not Score-Based

The LLM decides which tool to call based on query type, not confidence scores. Predictable. Debuggable. Fast.

1Memory Check — "Have I seen this?" → search_memory
↓ not found
2RAG Search — "Is it in my docs?" → search_rag
↓ not found / stale
3Web Search — "Quick public lookup" → search_web
↓ needs deep content
4Crawl — "Systematic extraction" → crawl_web

Crawling is always explicit and intentional — never auto-triggered.

Getting Started

Installation

Full Stack (Recommended)

git clone https://github.com/
  MasihMoafi/A-Modular-Kingdom
cd A-Modular-Kingdom
uv sync
cp config.example.toml config.toml
# edit: set embedding provider
uv run python -m src.host

Package Only

# No PyTorch (Ollama)
pip install rag-mem

# With local embeddings
pip install rag-mem[local]

# OpenAI embeddings
pip install rag-mem[openai]

Note: rag-mem[local] installs PyTorch (~2GB). Ollama provider recommended for production.
