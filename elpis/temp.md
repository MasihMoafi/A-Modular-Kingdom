
## Brainstorming & Principles (Added July 10)
- **The First Step of Execution:** For any project or idea, the first step MUST always be stating a clear definition of that thing. Right now, we lack a crystal-clear definition of Elpis.
- **Dynamic Context Management:** Since every API call is a fresh context window, we can use an active AI (or deterministic rules) to continuously scrub useless context, repetition, and terminal bloat. Memory isn't the context killer; agent chatter and terminal outputs are.
- **Simplicity is the Ultimate Sophistication:** Do not overload agents (e.g., 30+ tools in one MCP server is bad). Keep the OS minimal. However, do not force LLMs to be unnaturally silent, as suppressing their verbosity degrades their reasoning performance.
- **Signal Extraction:** Implement a feature in Elpis where an LLM extracts the pure signal/action items from a noisy or distracted user input.
